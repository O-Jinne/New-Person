# 🧑 새사람 프로젝트 — 운동 트래커 앱

Day 1~4 루틴을 선택하고, 각 운동을 체크하고, 실제로 친 무게/세트를 기록하고,
필요하면 커스텀 운동도 추가할 수 있는 개인 웹앱이야.

## 로컬에서 미리보기

`index.html` 파일을 더블클릭해서 브라우저로 열면 바로 확인 가능해.
(단, 이 상태에서는 아직 인터넷 어디서든 접속은 안 되고 이 컴퓨터에서만 보여)

---

## GitHub Pages로 무료 배포하기 (단계별)

### 1단계: GitHub 계정 만들기
- https://github.com 접속 → Sign up
- 이미 계정 있으면 스킵

### 2단계: 새 저장소(repository) 만들기
1. GitHub 로그인 후 우측 상단 `+` → `New repository`
2. Repository name: `monster-project` (원하는 이름 아무거나 OK)
3. `Public` 선택 (Pages 무료 사용하려면 Public이어야 함)
4. `Create repository` 클릭

### 3단계: 파일 업로드
1. 방금 만든 저장소 페이지에서 `uploading an existing file` 링크 클릭
2. 이 폴더 안의 3개 파일을 전부 끌어다 놓기:
   - `index.html`
   - `style.css`
   - `app.js`
3. 하단에 `Commit changes` 클릭

### 4단계: GitHub Pages 활성화
1. 저장소 상단 메뉴에서 `Settings` 클릭
2. 왼쪽 메뉴에서 `Pages` 클릭
3. `Source`를 `Deploy from a branch`로 설정
4. `Branch`를 `main` (또는 `master`), 폴더는 `/ (root)` 선택
5. `Save` 클릭

### 5단계: 완료!
- 몇 분 기다리면 상단에 초록색 박스로 URL이 뜸
- 형식: `https://[본인깃허브아이디].github.io/monster-project/`
- 이 URL을 아이폰 사파리에서 열고 "홈 화면에 추가" 하면 앱처럼 사용 가능

## ⚠️ 중요: 데이터 저장 방식

지금 버전은 **브라우저 localStorage**를 사용해.
즉, 데이터가 **이 기기의 이 브라우저에만** 저장돼.

- 아이폰 사파리에서 저장한 기록은 아이폰 사파리에서만 보임
- 크롬으로 열면 기록이 안 보임 (다른 저장소라서)
- 사파리 데이터/방문기록 삭제하면 기록도 같이 삭제됨

**여러 기기에서 동기화하고 싶으면** (예: 아이폰 + 컴퓨터 둘 다) 나중에
Firebase 같은 클라우드 데이터베이스를 연결해야 해. 이건 다음 단계로 확장 가능!
